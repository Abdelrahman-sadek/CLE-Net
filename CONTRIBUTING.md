## üìã Contribution Guidelines

### 1. Start with Discussion

Before writing code or making significant changes:

1. Check existing issues and discussions
2. Open a new issue to describe your proposed change
3. Discuss the approach with maintainers
4. Get agreement before implementing

### 2. Fork and Clone

```bash
git clone https://github.com/YOUR_USERNAME/cle-net.git
cd cle-net
```

### 3. Create a Feature Branch

```bash
git checkout -b feature/your-feature-name
```

### 4. Make Changes

Follow these coding standards:

- **Python**: PEP 8, type hints required
- **Documentation**: Markdown, clear explanations
- **Tests**: Required for new features
- **Commits**: Descriptive messages, reference issues

### 5. Submit a Pull Request

1. Push your branch to your fork
2. Open a PR against the main repository
3. Describe your changes clearly
4. Link related issues
5. Respond to feedback

## üß† Architecture Contributions

CLE-Net is an architecture-first project. When contributing to the core design:

- Explain the problem you're solving
- Show how your solution fits the existing mental model
- Consider edge cases and failure modes
- Discuss trade-offs explicitly
- Reference research or prior art
