# Frequently Asked Questions

## ü§î What is CLE-Net?

CLE-Net stands for **Decentralized Cognitive Agent Network**. It's an experimental architecture for AI agents that:

1. Extract symbolic laws from human interaction
2. Discover latent rules through symbolic regression
3. Persist knowledge across a decentralized network
4. Survive failures through distributed coordination

In short: CLE-Net turns conversations into cognitive laws.

---

## üß† What does CLE stand for?

**CLE = Cognitive Logic Extraction**

It refers to the process of converting unstructured human interaction (text, voice, documents) into symbolic representations and discovering the implicit rules that govern behavior.

---

## üåê Why a decentralized network?

Centralized AI systems have fundamental weaknesses:

- Single point of failure
- Dependent on operators
- Vulnerable to shutdowns
- Controlled by entities

CLE-Net distributes cognition so that:

- No single node controls memory
- The system survives individual failures
- Knowledge persists across machines and time
- No single entity can kill the network

---

## üîë What is Proof of Cognition (PoC)?

Proof of Cognition is CLE-Net's consensus mechanism. It replaces:

- **PoW**: Energy-heavy computations
- **PoS**: Capital-based voting

With:

**Independent cognitive discovery**

A rule achieves consensus when multiple independent agents, operating on different data, converge on the same symbolic rule. Truth emerges from independent reasoning, not authority or energy expenditure.

---

## ü§ñ How is this different from AI agents?

| Aspect | Traditional AI Agents | CLE-Net |
|--------|----------------------|---------|
| Focus | Answer questions | Discover laws |
| Memory | Local/context | Distributed/persistent |
| Reasoning | Statistical | Symbolic + statistical |
| Control | Centralized | Decentralized |

CLE-Net is not a chatbot or task executor. It's cognitive infrastructure.

## ‚öñÔ∏è Is CLE-Net a blockchain?

CLE-Net uses blockchain concepts but is not a blockchain in the traditional sense.

**CLE-Net uses blockchain for:**

- State persistence
- Consensus coordination
- Incentive mechanisms

**CLE-Net does NOT:**

- Focus on financial transactions
- Require proof-of-work
- Depend on specific chain (platform-agnostic)

## üéØ What is the economic model?

Still being designed. Current concepts:

- **Mining = Thinking**: Nodes earn rewards for discovering useful rules
- **Rule stability**: Rules that survive longer earn more
- **Simplicity bonus**: Simple, general rules preferred

Exact mechanisms require more research and community input.

## üîí How secure is CLE-Net?

CLE-Net has an explicit threat model (see [SECURITY.md](SECURITY.md)):

**Protected against:**
- Single malicious agents
- Rule spam
- Some Sybil attacks
- Data leakage (through hashing)

**Limitations:**
- Coordinated collusion may temporarily bias rules
- Global shutdown kills everything (physics)
- Economic model is experimental

Honest limitations are documented in `/docs/architecture/threat_model.md`.
