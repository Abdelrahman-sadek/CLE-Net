# CLE-Net Contributor Roles

## 1. Role Definitions

### 1.1 Core Developer

**Responsibilities**:
- Implement protocol changes
- Maintain codebase quality
- Review pull requests
- Ensure security

**Requirements**:
- Proven track record
- Deep understanding of architecture
- Active participation

### 1.2 Researcher

**Responsibilities**:
- Propose improvements
- Analyze security properties
- Write whitepaper sections
- Validate theoretical claims

**Requirements**:
- PhD or equivalent experience
- Publications in relevant fields
- Technical rigor

### 1.3 Validator

**Responsibilities**:
- Run validation nodes
- Participate in consensus
- Challenge invalid state transitions
- Report anomalies

**Requirements**:
- Technical competency
- High CCS score
- Economic stake

### 1.4 Community Member

**Responsibilities**:
- Provide feedback
- Report bugs
- Suggest improvements
- Vote on governance proposals

**Requirements**:
- Active participation
- Good faith engagement
- Technical interest

---

## 2. Role Evolution

Roles are not permanent:

- Contribution level determines role
- Inactivity can lead to role loss
- Exceptional contribution can accelerate advancement

---

## 3. Recognition

Contributors are acknowledged in:
- README contributors section
- Release notes
- Documentation credits
